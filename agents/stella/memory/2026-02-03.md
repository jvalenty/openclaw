
## Permission System Design (Complete)

Spent morning with John designing the Stellabot permission system. Key decisions:

**Human Roles (4-tier):**
- org_admin: full control, can create users/agents
- org_analyst: broad read for insights, can create reports
- org_editor: CRUD within scope, no user/agent creation
- org_user: use/collaborate within assigned scope

**Agent Roles (3-tier):**
- manager: can delegate, can spawn sub-agents
- specialist: domain expert, no delegation
- worker: temporary, scoped execution

**Core Model:**
- Role = capability ceiling
- Assignments = specific resources with CRUD+A flags
- Constraints = JSONB decorations for domain-specific rules (thresholds, states, time windows)
- Actions table = audit + approval queue + grease data

**Key Principles:**
- Neither human nor agent acts unilaterally on sensitive ops
- Human-to-human delegation forbidden (prevents sprawl)
- Agent delegation only for parallel throughput by managers
- Grease earned from successful patterns (auto-approve after 10+ clean executions)

**Stress tested against:**
- Prop trading firm hierarchy ✓
- Content distribution network ✓

**Lesson learned:** Don't settle for "made it work" — think harder for the right design.
