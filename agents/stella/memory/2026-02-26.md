## ⚠️ SECRET EXPOSURE INCIDENT (19:08 PST)
**What happened:** Used `fly ssh console --command "wget ... /files/read"` to test machine reachability from Fly. The command returned the full `openclaw.json` content including all secrets (ANTHROPIC_API_KEY, OPENAI_API_KEY, Slack tokens, Telegram bot token, ElevenLabs key, gateway token, skill API keys).

**Why it happened:** Testing if token fix worked — should have just checked HTTP status code, not read the file content.

**Correct approach:**
```bash
# Test reachability: check status ONLY
STATUS=$(curl -s -o /dev/null -w "%{http_code}" -X POST https://stella.e2e.pro/files/read \
  -H "Authorization: Bearer $TOKEN" -d '{"path":"/tmp/test"}')
echo "Status: $STATUS"  # 401=wrong token, 200=works, never echo file contents
```

**Keys exposed (need rotation):** All keys in ~/.openclaw/openclaw.json — John to schedule controlled rotation.

**Rule reinforced:** NEVER use commands that return file contents in test scenarios. Always use `-o /dev/null -w "%{http_code}"` pattern for reachability checks.
