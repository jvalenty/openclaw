## Root Causes Found (from actual logs)
Three compounding bugs in Machine Service browser session handling:

1. **Wrong pool key** — `executeLoginSteps` in `browser-auth-api.ts` sent only `X-Org-Id`, no `X-Agent-Id`. Machine Service keys pools by `org+agent`. Login used the org-level pool (profile: `.../22377b4f/`), all other tools used the agent pool (profile: `.../22377b4f/26187726-399c.../`). Different profiles = different cookies = second window always hit auth redirect.

2. **Cookies never persisted to disk** — After `browser_login` succeeded, cookies lived in memory only. Every browser crash (stale context) restarted the pool from a clean profile. Earnware session gone.

3. **Stale context crashes** — Browser dying constantly (`"Target page, context or browser has been closed"`). Each crash = pool restart = fresh context = no cookies.
