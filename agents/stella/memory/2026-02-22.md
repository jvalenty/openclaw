## Multi-Agent Research (Critical Finding)
- **Two gateways sharing one Socket Mode app token = BROKEN** — Slack load-balances events between connections, so only one gateway receives each message
- **The right way:** Multiple agents on ONE gateway with `bindings` and Slack `accounts`
- OpenClaw docs: https://docs.openclaw.ai/concepts/multi-agent
- YouTube builders run teams of agents on a single gateway, not separate machines
- `tools.agentToAgent` enables direct inter-agent messaging on same gateway
- For our case (Stella + Bella on separate machines), options:
  1. Add Bella as second agent on Stella's gateway (cleanest)
  2. Two separate Slack apps (one per gateway)
  3. HTTP mode with webhook fan-out (complex)
