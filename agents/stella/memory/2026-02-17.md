### Investigation Completed

#### 1. Feb 11 Bricking Root Cause
- Modified `~/.clawdbot/clawdbot.json` gateway.bind setting
- Confused Machine Service (18900) with Clawdbot Gateway (18789)
- Broke ability to receive messages for 70+ minutes
- John had to manually restore config

#### 2. Update Path Options
**Migration**: `npm install -g openclaw@latest` → `openclaw doctor` → restart
- Doctor handles config migration (~/.clawdbot → ~/.openclaw)
- Risk: Config format changes could cause issues

**Clean Rebuild**: Fresh `openclaw onboard --install-daemon`
- Loses session history but cleaner
- Memory files safe (in workspace, not state dir)

### Key Learnings

**What goes in git (workspace):**
- SOUL.md, AGENTS.md, USER.md, TOOLS.md, HEARTBEAT.md
- memory/*.md files
- Scripts

**What stays OUT of git:**
- ~/.openclaw/openclaw.json (config)
- ~/.openclaw/credentials/ (API keys, OAuth)
- Session transcripts
- Any secrets

**Recommended Structure:**
```
jvalenty/openclaw/
├── agents/
│   ├── stella/
│   │   ├── SOUL.md, AGENTS.md, USER.md, TOOLS.md, HEARTBEAT.md
│   │   └── memory/
│   └── bella/
│       └── (same structure)
├── shared/
│   └── GLOBAL_RULES.md
├── scripts/
│   └── sync.sh, consolidate.py
└── .gitignore
```
