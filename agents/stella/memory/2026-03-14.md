## Browser Pool Root Cause — popup interceptor race (16:00-16:30 PDT)

Dan was down all day. Final root cause was NOT profile corruption or singleton locks —
those were symptoms. The **actual bug**: `context.on('page')` in Playwright fires for ALL
new pages, including our own `context.newPage()`. The popup interceptor was unconditionally
calling `newPage.close()` after `waitForLoadState('domcontentloaded')` — which resolves
INSTANTLY for `about:blank`. Race: interceptor closed our page before `addInitScript` ran.

**Fix (PR #11, deployed 23:07 UTC):**
- Popup interceptor returns early if URL is `about:blank` or `about:*`
- Added `--no-first-run` / `--no-default-browser-check` Chrome flags

**Lesson:** Never unconditionally close pages in `context.on('page')`. Always check URL first.
Profile quarantine (PR #8) and singleton sweep (PR #9) are real improvements but were
not the root cause of today's outage.
