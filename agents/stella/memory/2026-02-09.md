## Architecture Decisions - FINAL (evening session)

### Core Philosophy
**Stop building infrastructure. Let the model be the solution.**

The agent is smart. Give it:
1. Context (skills, conversation history)
2. Memory (short/medium/long term)
3. Way to stay active (cron + callbacks)

Then let it work like a human team member.

### Communication Architecture

**Stellabot chat = Primary interface**
- All soft agent communication
- Group threads for agent + human collaboration
- @mentions for attention
- Push notifications via PWA

**Telegram = Mac Mini backdoor only**
- Direct hardware/sys admin access
- Emergency access if Stellabot is down
- Keep this channel, but not for primary work

### Agent Model

**Soft agents (Stellabot):**
- Primary workers, directors
- Get tasks same way humans do (not special job queues)
- First class team members
- Stay active via cron wake-ups + callbacks

**Mac Mini (Clawdbot):**
- Hardware access when needed
- Browser automation for persistent sessions
- Sys admin tasks
- Called by soft agents or directly via Telegram

### What We DON'T Need
- ❌ Job queues
- ❌ Agent inbox tables
- ❌ Worker status tracking
- ❌ Complex orchestration
- ❌ Special notification channels
- ❌ n8n-style workflow automation

### What We DO Need
- ✅ Cron mechanism for soft agent wake-ups
- ✅ Callback handling when Mac Mini finishes work
- ✅ Push notifications in Stellabot PWA
- ✅ Good @mention support in group threads
