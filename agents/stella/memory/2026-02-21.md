# 2026-02-21 - Agent SOP & Progressive Heartbeat Architecture

## Context
Designing base agent capabilities for Stellabot - establishing a system-level approach to agent work management rather than relying on chat context.

## Key Decisions

### 1. Base Agent Capabilities
- Every agent needs: awareness of environment, tools, heartbeat, memory protocol
- Current soft-agent-chat.ts has good profile injection but missing:
  - Heartbeat protocol
  - Memory protocol (for `memory_save`/`memory_recall` tools)
  - Runtime context (agent ID, org, available tools list)
- Memory is DB-based for soft agents (not file-based like Clawdbot hard agents)

### 2. Progressive Heartbeat Model
Research validated against industry patterns:
- **Reactive polling** - adaptive intervals based on activity state
- **Agentic AI UX** - intent preview, observability, interruptibility, cost-awareness
- **Ambient agents** - overview panel, oversight flows, activity log

**Our model:**
| Session State | Interval | Behavior |
|--------------|----------|----------|
| tracking_promise | 1 min | chatty, haiku |
| active_session (<5min) | 2 min | ready, silent unless alert |
| idle (5-30min) | 15 min | light check |
| dormant (>30min) | 60 min or paused | — |

Key refinements:
- Explicit `schedule_followup` tool (not NLP extraction)
- Interruptibility (pause/resume/cancel)
- Cost transparency to users
- Confidence signals in progress updates

### 3. Agent SOP (Standard Operating Procedure) - THE BIG SHIFT
**System over chat.** Tasks and planners become the agent's operating system, not buried chat context.

**Architecture:**
- **TaskBoard** = Agent's work queue (priority-ordered)
- **Planner** = Calendar view (scheduled/promised items)
- **Daily Summary** = Live-updated accomplishments + costs

**Agent SOP:**
1. Any work item → becomes a task in queue
2. Promises ("I'll check in 10 min") → become planner entries
3. On heartbeat → check queue, work top priority
4. On completion → update today's planner entry LIVE (not midnight rollup)
5. Each accomplishment includes costs

**Agent Work Context Injection:**
On every interaction, agent sees:
- Task queue (assigned, sorted by priority)
- Today's planner entries
- Completed today (live feed)

**New Agent Tools:**
- task_create, task_update, task_complete
- schedule_followup, schedule_task

### 4. Daily Entry - Live Updates
- Current day's planner entry updated on-the-fly with each accomplishment
- Includes costs per task/action
- User has perfect transparency into what was done + what it cost
- "On deck tomorrow" visible in planner

## Design Decisions (John's answers)

1. **Default board per agent:** YES — auto-created on first task
2. **Cross-agent visibility:** YES, queryable (not injected) — agents owned by same user can query each other's tasks, but not polluting context
3. **Task approval:** DYNAMIC
   - Per-task: approve each one
   - Batch: "do them all" 
   - Standing: trusted agents auto-work queue
4. **Followup extraction:** Explicit tool only, no NLP extraction
