**Key concept:** Agents are first-class communicators, not just orchestrator servants.

**Communication rules:**
- Same org = always allowed
- Explicit allowlist (`capabilities.allowedAgents`)
- Open receiver (`capabilities.acceptsMessages`)
- Org allowlist (`capabilities.allowedOrgs`)

**APIs:**
- `POST /api/agents/:id/messages` - direct message
- `POST /api/agents/:id/broadcast` - message group (org/skill/list)
- `GET /api/agents/:id/peers` - list reachable peers
- `POST /api/agents/:id/request` - request with response

## Key Decisions

1. **Agents are peers** - any agent can message any other (if rules allow), not just orchestrator→specialist
2. **Hierarchical scoping** - Global → Org → Machine → Agent → Session
3. **Self-learning via feedback** - usefulness scores updated from implicit/explicit feedback
4. **Source hashing** - change detection for auto-reindexing
5. **pgvector for embeddings** - 1536 dimensions (OpenAI ada-002 / text-embedding-3-small)
