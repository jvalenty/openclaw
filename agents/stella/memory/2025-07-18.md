
## Earnware Pub Orders Workflow (Checkpoint)

**QA Score Rules:**
- Score **≥75** = Good to go, proceed with editing/publishing
- Score **<75** = Trouble — usually means:
  - Story too new, researcher couldn't find enough current sources
  - Some other research/generation failure
  
**Best Practice:**
- When doing feed selections, run **extra stories as buffer** (more than you need)
- This accounts for failures and gives you options
- Stories under 75 can be fixed manually if needed, but better to have backups

**Workflow:** Feed → Bookmark → Assign Writers → Pub Orders → Review (≥75 only) → Edit/Publish
