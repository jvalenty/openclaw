## Critical Fixes
- **Anthropic API Key Staleness**: John noticed my Anthropic API key was pulling from an old key (ending in `fQAA`) instead of the new Stellabot key (`GwAA`). 
- **Root Cause**: The old key was hardcoded in `~/Library/LaunchAgents/ai.openclaw.gateway.plist` under `EnvironmentVariables`, which completely overrode any OpenClaw config updates.
- **Resolution**: I used a Python script with `plistlib` to cleanly remove the `ANTHROPIC_API_KEY` entry from the LaunchAgent plist, and then reloaded the service via `launchctl`. OpenClaw is now properly pulling the key from Stellabot's injected configuration.
