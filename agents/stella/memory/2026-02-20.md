## Reseller Architecture - Phase 4 Decisions

Reviewed full spec `docs/specs/reseller-architecture.md` with John. Key decisions:

### Billing Granularity
- **Hourly cron** for usage aggregation, not per-request calcs (overhead)
- Request path just checks `billing_hold` flag (fast)
- "Can't get in too much trouble" in an hour

### Degraded Before Hard Block
- 80% of limit → degrade to Haiku + persistent chat banner
- 100% of limit → hard 403
- Manual override available for trusted customers
- Same pattern as agent-level cost controls

### Authorization Inheritance
- **No** — each org needs explicit authorization
- Sub-orgs don't inherit parent's machine access

### Org-Level OAuth
- Keep as fallback for now (small orgs sharing one Google account)
- New agents get individual creds
- Can deprecate org-level later once all agents have individual auth

### Self-Service Authorization
- **Yes** — tied to product packages
- Customer can self-serve request access
- Package determines value/cost assignment
- Not just reseller-initiated

### Conflicts
- Non-issue. Spec is clear: agent secrets from owning org, everything else from target org

## Bug Fix: myRole undefined in org switcher (12:47 PM)

## Exec Output Compression Spec (10:00 PM)

Created spec for token-saving feature inspired by RTK (github.com/rtk-ai/rtk).

**Location:** `~/e2e/stellabot/docs/specs/exec-output-compression.md`

**Key Features:**
- Automatic output compression at Machine Service exec layer
- 60-90% token savings on common dev commands
- Compressors for: git, test, build, ls, logs, package managers
- File reading modes: signatures, structure, relevant
- Savings tracking with analytics dashboard

**Compression Modes:**
- `auto` — detect command type, apply appropriate compressor
- `minimal` — aggressive compression
- `none` — passthrough (opt-out)

**Implementation Phases:**
1. Core infrastructure + git compressor
2. Test/build/lint compressors
3. File reading compression
4. Analytics & UI

Estimated 4 weeks to full implementation.

## Exec Output Compression - Phase 1 Complete (10:20 PM)

Implemented token-saving compression for Machine Service exec output.

**Compressors built:**
- `git` — status, push, pull, log, diff, commit (75-92% savings)
- `test` — jest, vitest, pytest, cargo test, go test (failures only)
- `build` — tsc, eslint, cargo, go vet (errors/warnings only)
- `files` — ls, find, tree (grouped, icons)
- `package` — npm, yarn, pip, cargo install (strip progress)

**Verified savings:**
- `git status`: 260 → 64 chars (75%)
- `ls -la`: 2,842 → 471 chars (83%)

**Key design decisions:**
- Errors always preserved in full
- Success paths compressed aggressively
- Agent can opt-out with `compress: "none"`
- Stats available via `includeStats: true`

**Location:** `~/e2e/machine/src/utils/compressors/`
