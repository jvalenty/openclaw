# 2026-02-18

## Cost Controls & Model Management — Design Checkpoint

### Core Concept
Three-layer cost controls (System → Org → Agent) with smart model selection.

### Model Selection Modes (UI dropdown below chat input)
- **Auto**: Mini classifies task, routes to optimal model
- **Fast**: Always Haiku (mini)
- **Smart**: Always Sonnet (standard)  
- **Genius**: Always Opus (frontier)

### Hierarchy
```
SYSTEM: Available models, pricing, global limits
  └─ ORG: Enabled models, cost limits, default agent limits
      └─ AGENT: Allowed modes, limits (≤org), degraded state
```

### Key Features
1. **Usage tracking**: Every request logged with tokens, cost, model, mode
2. **Limit enforcement**: Pre-request check, auto-downgrade when exceeded
3. **Degraded mode**: Forces mini, persistent banner in chat, auto-resets at period boundary
4. **Smart classifier**: Mini analyzes complexity (0-1), reasoning depth, code needs → routes
5. **Admin notifications**: Alert at 80%, on degraded, on spend spikes
6. **BYOK support**: Orgs can bring own API keys

### Schema Additions
- `model_registry` — system model catalog with pricing
- `orgs.model_config` — enabled models, priorities, allowed modes
- `orgs.cost_limits` — daily/monthly caps, alert thresholds
- `agents.model_config` — allowed modes, model override
- `agents.cost_limits` — per-agent caps (inheritable from org)
- `agents.cost_state` — degraded flag, current spend, reset time
- `usage_records` — per-request logging for aggregation

### Implementation Phases
1. Foundation: Tables, usage recording
2. Limits: Budget checks, auto-downgrade, degraded state
3. Smart Routing: Mode selector UI, classifier, router
4. Admin UI: Dashboards, config pages, degraded banner
5. Notifications: In-app, email, webhooks

## Final Design Checkpoint (23:03 PST)

**Full spec committed:** `docs/specs/cost-controls-model-management.md`

### Architecture Decision
**Accounting tables are source of truth** for billing. Analytics (usage_summaries) deferred to later phase.

### Core Tables (priority order)
1. `model_registry` — Model catalog with pricing
2. `token_accounts` — Per-org billing account, Stripe link
3. `token_transactions` — Immutable ledger (source of truth)
4. `billing_periods` — Invoice cycles for postpaid
5. `credit_packages` — Purchasable bundles
6. `usage_records` — Raw per-request log
7. `cost_alerts` — Admin notifications
8. `usage_summaries` — Pre-aggregated analytics (later)

### Billing Modes
| Mode | Description |
|------|-------------|
| `demo` | Free trial credits (default new orgs) |
| `prepaid` | Buy credits via Stripe |
| `postpaid` | Monthly invoice |
| `byok` | Bring own API keys |
| `unlimited` | Sys admin override |

### Overrides
- **Sys override**: Admin sets unlimited/custom limits on any org
- **Org override**: Org overrides agent defaults

- 001-006: Model registry, org/agent config, usage, alerts, summaries
