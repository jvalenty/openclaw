## Schema Refinements (Campaigns vs Projects vs Workflows)
- **New Mental Model (Refined 15:28 - 15:49):**
  - `Workflow` = Deterministic data pipelines (Scrape -> Enrich -> LLM -> Build Demo Newsletter Project). This does the heavy lifting.
  - `Campaign` = Outreach / Distribution (Email, Social, Ads).
  - `Project` = Actual client deliverable (e.g., the generated prototype or a marketing funnel for XYZ brand).
- **Execution Engine:** Webmatic `jobs` and `workers` are mothballed for this use case because they are modeled after synthetic human personas. Instead, long-running scrapes and data enrichments belong to a **Workflow**, which is offloaded to **Trigger.dev** for execution and webhooked back to Webmatic's `contacts` table.
- **Contact Centricity:** `Contact` is the primary entity. Add an `enrichment_data` or `brand_assets` JSON block to the `contacts` table.
