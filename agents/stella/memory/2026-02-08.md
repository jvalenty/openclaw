## Major Accomplishments

### 1. Soft Agent Architecture Proven
Successfully created and tested "Stella Soft" - a soft agent that runs via Claude API directly in Stellabot while using hardware tools through Clawdbot.

**Key insight:** Agents don't need to RUN on Clawdbot to USE Clawdbot's tools. Clawdbot becomes a "tool server" rather than an "agent runtime."

### 2. Agent Routing Logic Documented

**Hard Agent (routes to Clawdbot):**
- `agentType === 'sys_admin'` OR
- Has `machineId` set OR
- `agentId.startsWith('machine:')`

**Soft Agent (runs via Claude API in Stellabot):**
- Not sys_admin
- No machineId
- No direct endpoint
- Routes through `soft-agent-chat.ts`

**Code location:** `server/routes/agent-chat.ts` lines 377-385

```javascript
const isSoftAgent = agent && 
  agent.agentType !== 'sys_admin' && 
  !agent.machineId &&
  !agent.endpoint &&
  agentId !== 'stella' && 
  agentId !== 'main' &&
  !agentId.startsWith('machine:');
```

### What's Working

1. **Soul Loading:** `modelConfig.systemPrompt` → injected into context
2. **Knowledge Loading:** `buildSystemPrompt()` queries `knowledge` table:
   - `active = TRUE`
   - `approved = TRUE`
   - Scoped to system / org / agent
   - Ordered by priority, limited to 10
3. **Skills Loading:** Matches by agent ID or slugified agent name
4. **User Context:** Includes user name/role in prompt

### What's Missing

1. **Memory Write Tools** - Agents can't save learnings
2. **Lesson Extraction** - No auto-extraction from conversations
3. **Daily Logs** - No `memory/YYYY-MM-DD.md` equivalent
4. **Long-term Memory** - No curated `MEMORY.md` equivalent
5. **Semantic Search** - No RAG/vector search over knowledge

## Memory Tools Implementation Complete! ✅

**Commit:** `0497b5a` - feat: add memory_save and memory_recall tools for soft agents

Added 163 lines to `server/routes/soft-agent-chat.ts`:

### 1. Tool Definitions
- `MEMORY_SAVE_TOOL` - Save facts, decisions, lessons, preferences, rules, notes
- `MEMORY_RECALL_TOOL` - Search memories by query with optional type filter
- `MEMORY_TOOLS` array combining both

### 2. Permission Mapping
```typescript
'memory_save': 'memory.write',
'memory_recall': 'memory.read',
```

### 3. Tool Availability Check
Memory tools loop in `getAvailableTools()` (between sandbox and browser checks)

### 4. Execution Logic

**memory_save:**
- Inserts into `knowledge` table with `scope='agent'`
- Auto-approved (`active=TRUE`, `approved=TRUE`)
- Importance mapped to priority: critical=100, high=75, normal=50, low=25
- Supports tags array and expires_days

**memory_recall:**
- ILIKE text search on title + content
- Filters by agent_id and optional type
- Ordered by priority DESC, created_at DESC
- Default 5 results, max 20

### To Enable for an Agent
Grant permissions in `agent_actions` table:
```sql
INSERT INTO agent_actions (id, agent_id, action_type, enabled)
VALUES (gen_random_uuid(), '<agent-id>', 'memory.write', true),
       (gen_random_uuid(), '<agent-id>', 'memory.read', true);
```
