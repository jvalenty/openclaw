# 2026-02-17 — Machine Service Mesh Architecture

## Key Decisions Made Today

### 1. Machine Service Runs on Host (Not OrbStack)
- Needs hardware access: browser, screen, camera
- OrbStack can't see host desktop or access cameras
- PTY works fine on bare metal
- Simpler architecture

### 2. Soft Agents Operate THROUGH Machine Service
- No agent needed on machines
- Machine Service is a "dumb" tool server — no reasoning
- All decisions happen in Stellabot (soft agents)
- Machines are interchangeable

### 3. Local Agent Runtime KILLED 🪦
John's question: "Why would I want an agent for interactive dev work if soft agents can do everything through Machine Service?"

Answer: You don't. With PTY support:
- Soft agent → Machine Service PTY → spawn "claude" → drive Claude Code CLI
- Gets Max Plan cost benefit without separate runtime
- One architecture, not two
- Deprioritized, code exists at `~/e2e/agents/` but not maintained

### 4. PTY is the Critical Unlock
Enables:
- Long-running ops (sessions survive disconnect)
- Claude Code CLI as a soft agent tool
- Interactive SSH, vim, etc.
- Completes Clawdbot parity

## Machine Mesh Vision

3 Mac Minis, each running Machine Service, zero agents on them.
Soft agents in Stellabot orchestrate everything remotely.

```
STELLABOT (orchestrator)
       │
       ├── Mini 1 (Machine Service)
       ├── Mini 2 (Machine Service)  
       └── Mini 3 (Machine Service)
```

## Machine Routing Implementation (2026-02-17)
Added explicit machine routing to soft agent tools:

**Tools updated:**
- pty_spawn, pty_list, pty_read, pty_write, pty_kill
- browser_navigate, browser_snapshot, browser_screenshot, browser_click, browser_type, browser_scroll, browser_wait, browser_login, browser_new_tab, browser_close_tab
- local_read, local_write, local_edit, local_list
- process_start, process_list, process_status, process_input, process_kill

**New tools:**
- machines_list - discover available machines by capability

**How it works:**
1. If `machine` param provided → look up machine by ID/name in DB
2. Check org access (same org, platform machine, or no org)
3. Build service URL from machine's ipAddress + machineServicePort
4. Use machine's machineServiceToken for auth
5. If no `machine` param → fall back to MACHINE_SERVICE_URL env var

### Console Proxy Fixes
Two issues found and fixed:

**1. X-Frame-Options blocking iframe (commit 4dc822d)**
- Helmet middleware set `X-Frame-Options: DENY` on all responses
- Fixed: Override headers in console proxy to allow SAMEORIGIN
- Added `frame-ancestors 'self' https://stellabot.app`

**2. Wrong service URL for Clawdbot machines (commit b6aa3ba)**
- "Stellabot Mac-Mini" has tunnel_url pointing to Clawdbot (18789)
- Console needs Machine Service (18900)
- Fixed: Prefer `ip_address:machine_service_port` over `tunnel_url`
