## TTS Conversational Filter Feature (PENDING DEPLOY)

**Status:** Code complete, awaiting John's deploy approval

**What it does:** Optional toggle per agent that runs TTS output through Haiku to make it sound more natural when spoken (removes markdown, URLs, code blocks, rewrites for conversational flow).

**Files changed:**
- `~/e2e/stellabot/server/routes/tts-filter.ts` — NEW endpoint
- `~/e2e/stellabot/server/routes.ts` — registered route at `/api/tts-filter`
- `~/e2e/stellabot/client/src/pages/AdminAgentEdit.tsx` — added ttsFilter state + UI toggle in Voice Settings
- `~/e2e/stellabot/client/src/hooks/use-chat-audio.ts` — calls filter before TTS when enabled

**How it works:**
1. Agent Edit → Voice Settings → "Conversational Filter" toggle
2. Saves `ttsFilter: boolean` to agent's modelConfig
3. When TTS speaks, if filter enabled → POST /api/tts-filter with text
4. Endpoint runs through Haiku with prompt to make text natural for speech
5. Filtered text goes to ElevenLabs TTS

**Smart skipping:** Filter skips short text (<50 chars) or text that's already clean (no markdown/URLs/code).
